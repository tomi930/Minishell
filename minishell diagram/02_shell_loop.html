🔄 The Shell Loop (REPL)

The heartbeat of your minishell — it never stops looping until you tell it to

The Core Idea: Read → Evaluate → Print → Loop

Your shell is an infinite loop. It does the same 4 things over and over:

1

PROMPT

Display "minishell$ " and wait for the user to type something

2

READ

Get the line the user typed (using readline)

3

PARSE

Break the input into tokens, understand the structure

4

EXECUTE

Run the command(s), show output, then go back to step 1

In Pseudocode

int main(int ac, char **av, char **envp) { char *line; t_data data; init_shell(&data, envp); // setup env, signals while (1) // ← THE INFINITE LOOP { line = readline("minishell$ "); // 1. PROMPT + READ if (!line) // Ctrl+D → NULL → exit break; if (*line) add_history(line); // save for up-arrow recall tokens = lexer(line); // 2. TOKENIZE tree = parser(tokens); // 3. BUILD STRUCTURE executor(tree, &data); // 4. EXECUTE free_all(line, tokens, tree); // cleanup for next loop } return (data.exit_status); }
🎮 Game Loop Analogy: Think of a video game. Every frame: read input → update game state → render screen → repeat. A shell does the same thing: read command → process it → show result → repeat. The game loop runs at 60fps. Your shell loop runs at "human typing speed" fps!
