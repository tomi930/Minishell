🔗 Pipes: Connecting Processes

How "ls | grep .c" actually works at the system level

What Is a Pipe?

A pipe is a one-way data channel created by the OS. It has two ends: a write end (where data goes IN) and a read end (where data comes OUT). Data flows in one direction only, like a water pipe.

🚰 Water Pipe Analogy: Imagine a pipe connecting two rooms. Room A pours water in (write end). Room B collects water out (read end). Water can only flow one way. If nobody is pouring, the collector waits. If the pipe is full, the pourer waits. That's exactly how Unix pipes work!

⚠ THE #1 PIPE BUG: Forgetting to close unused pipe ends. If the parent keeps the write end open, the reading child will NEVER get EOF — it'll hang forever waiting for more data. Every process must close pipe ends it doesn't use!
